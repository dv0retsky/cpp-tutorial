# Тема №22.Реализация конструкторов разных типов

**Конструкторы** — это фундаментальная часть объектно-ориентированного программирования в C++. Как мы уже знаем, конструктор — это специальный метод класса, который автоматически вызывается при создании объекта. Его основная задача — инициализировать объект корректным и непротиворечивым состоянием.

Однако конструкторы могут быть разными. В зависимости от количества и типов передаваемых аргументов, а также цели использования, в C++ можно выделить несколько видов конструкторов. Умение правильно их реализовывать позволяет программисту гибко управлять процессом создания объектов: от простой инициализации значениями по умолчанию до создания точных копий сложных структур данных.

## Виды конструкторов

В C++ существует несколько типов конструкторов, которые отличаются способом вызова и назначением.

4.	**Конструктор по умолчанию (Default constructor)** — конструктор, который можно вызвать без аргументов. Если программист не определяет ни одного конструктора, компилятор пытается сгенерировать свой собственный конструктор по умолчанию.
5.	**Конструктор с параметрами (Parameterized constructor)** — конструктор, который принимает один или несколько аргументов. Он используется для инициализации объекта конкретными значениями.
6.	**Конструктор копирования (Copy constructor)** — конструктор, который создает новый объект как копию существующего объекта того же класса. Он принимает один аргумент — ссылку на объект того же класса (обычно константную).
7.	**Конструктор перемещения (Move constructor) (изучается позже)** — конструктор, который «перемещает» ресурсы из одного объекта в другой, не копируя их.

Рассмотрим класс `Book`, который будет демонстрировать работу этих конструкторов.

### ПРИМЕР 1 (Конструктор по умолчанию)

Конструктор по умолчанию не принимает аргументов. Он инициализирует объект значениями по умолчанию.

```cpp
#include <iostream>
#include <string>

class Book {
private:
    std::string title;
    int pages;
    double price;

public:
    // Конструктор по умолчанию
    Book() {
        title = "Неизвестно";
        pages = 0;
        price = 0.0;
        std::cout << "Вызван конструктор по умолчанию. Книга: " << title << std::endl;
    }

    void displayInfo() const {
        std::cout << "Название: \"" << title << "\", Страниц: " << pages
                  << ", Цена: " << price << " руб." << std::endl;
    }
};

int main() {
    Book myBook; // Создание объекта с помощью конструктора по умолчанию
    myBook.displayInfo();
    return 0;
}
```

### ПРИМЕР 2 (Конструктор с параметрами)

Конструктор с параметрами позволяет задать начальные значения полей объекта непосредственно при его создании. Это основной способ инициализации в реальных программах.

```cpp
#include <iostream>
#include <string>

class Book {
private:
    std::string title;
    int pages;
    double price;

public:
    // Конструктор по умолчанию
    Book() {
        title = "Неизвестно";
        pages = 0;
        price = 0.0;
        std::cout << "Конструктор по умолчанию. Книга: " << title << std::endl;
    }

    // Конструктор с параметрами
    Book(std::string bookTitle, int bookPages, double bookPrice) {
        title = bookTitle;
        pages = (bookPages > 0) ? bookPages : 0; // Простая проверка
        price = (bookPrice > 0) ? bookPrice : 0.0;
        std::cout << "Вызван конструктор с параметрами. Книга: " << title << std::endl;
    }

    void displayInfo() const {
        std::cout << "Название: \"" << title << "\", Страниц: " << pages
                  << ", Цена: " << price << " руб." << std::endl;
    }
};

int main() {
    Book book1; // Вызов конструктора по умолчанию
    book1.displayInfo();

    // Вызов конструктора с параметрами
    Book book2("Война и мир", 1300, 950.50);
    book2.displayInfo();

    Book book3("Программирование на C++", -5, -100); // Проверка коррекции данных внутри конструктора
    book3.displayInfo();

    return 0;
}
```

### ПРИМЕР 3 (Конструктор копирования)

Конструктор копирования создает новый объект путем копирования всех полей из другого объекта. Если его не определить, компилятор создаст его автоматически (поверхностное копирование). Однако для классов, управляющих динамической памятью (как `SimpleArray` из прошлого занятия), необходимо писать свой конструктор копирования для глубокого копирования (deep copy).

```cpp
#include <iostream>
#include <string>

class Book {
private:
    std::string title;
    int pages;
    double price;

public:
    // Конструктор по умолчанию
    Book() : Book("Неизвестно", 0, 0.0) { // Делегирование конструктора (C++11)
        std::cout << "Конструктор по умолчанию (делегирование)." << std::endl;
    }

    // Конструктор с параметрами
    Book(std::string bookTitle, int bookPages, double bookPrice) {
        title = bookTitle;
        pages = (bookPages > 0) ? bookPages : 0;
        price = (bookPrice > 0) ? bookPrice : 0.0;
        std::cout << "Конструктор с параметрами. Книга: " << title << std::endl;
    }

    // Конструктор копирования
    Book(const Book& other) {
        title = other.title;
        pages = other.pages;
        price = other.price;
        std::cout << "Вызван конструктор копирования. Создана копия книги \"" << other.title << "\"." << std::endl;
    }

    void displayInfo() const {
        std::cout << "Название: \"" << title << "\", Страниц: " << pages
                  << ", Цена: " << price << " руб." << std::endl;
    }
};

int main() {
    Book originalBook("Оригинал", 500, 300.0);
    originalBook.displayInfo();

    // 1-й способ: прямая инициализация
    Book copyBook1(originalBook);
    copyBook1.displayInfo();

    // 2-й способ: инициализация через присваивание (copy-initialization)
    Book copyBook2 = originalBook;
    copyBook2.displayInfo();

    return 0;
}
```

### Списки инициализации
**Список инициализации (member initializer list)** — это синтаксическая конструкция, которая следует после списка параметров конструктора и начинается с двоеточия. Она позволяет инициализировать поля класса непосредственно перед выполнением тела конструктора. Это более эффективный и часто единственный способ инициализации константных полей, полей-ссылок и полей, не имеющих конструктора по умолчанию.

### ПРИМЕР 4

```cpp
#include <iostream>
#include <string>

class Book {
private:
    const std::string title; // Константное поле (должно быть инициализировано сразу)
    int pages;
    double price;

public:
    // Использование списка инициализации
    Book(std::string bookTitle, int bookPages, double bookPrice)
        : title(bookTitle), pages(bookPages), price(bookPrice) {
        // Тело конструктора теперь может быть пустым или содержать сложную логику
        std::cout << "Конструктор со списком инициализации. Книга: " << title << std::endl;
    }

    void displayInfo() const {
        std::cout << "Название: \"" << title << "\", Страниц: " << pages
                  << ", Цена: " << price << " руб." << std::endl;
    }
};

int main() {
    Book myBook("Эффективный C++", 350, 1200.0);
    myBook.displayInfo();
    return 0;
}
```

### Ключевое слово `explicit`

**Ключевое слово `explicit`** используется перед конструктором с одним параметром, чтобы запретить неявные преобразования типов. Иногда компилятор может автоматически преобразовывать тип аргумента в тип класса, что может привести к неожиданному поведению и ошибкам. `explicit` делает конструктор "явным".

### ПРИМЕР 5

```cpp
#include <iostream>

class Integer {
private:
    int value;

public:
    // Обычный конструктор. Позволяет неявное преобразование.
    // Integer(int v) : value(v) {}

    // Явный конструктор. Неявное преобразование запрещено.
    explicit Integer(int v) : value(v) {}

    int getValue() const { return value; }
};

void printInteger(const Integer& i) {
    std::cout << "Значение: " << i.getValue() << std::endl;
}

int main() {
    Integer i1(10); // Явный вызов - разрешен всегда
    // Integer i2 = 20; // ОШИБКА при explicit! Неявное преобразование int -> Integer

    printInteger(i1); // Ok
    // printInteger(30); // ОШИБКА при explicit! Нельзя неявно преобразовать 30 в Integer

    // При explicit конструкторе вызов должен быть явным:
    printInteger(Integer(40)); // Ok

    return 0;
}
```
