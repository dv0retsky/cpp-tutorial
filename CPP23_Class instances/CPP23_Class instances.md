# Тема №23. Динамическая память, создание, удаление экземпляров классов. Жизненный цикл объектов классов

В предыдущих занятиях мы создавали объекты классов простым способом: `ClassName objectName`. Такой способ называется **статическим созданием** — память под объект выделяется на стеке, и время жизни объекта автоматически управляется компилятором. Объект создается при входе в область видимости и уничтожается при выходе из нее.

Однако в реальной разработке часто возникает необходимость более гибкого управления памятью. Представьте, что вы разрабатываете систему для онлайн-кинотеатра. Количество одновременно просматривающих фильмы пользователей заранее неизвестно — оно меняется каждую секунду. Выделять память под максимально возможное количество зрителей статически — значит впустую расходовать ресурсы сервера. Гораздо эффективнее создавать объекты "зритель" динамически, по мере подключения пользователей, и уничтожать их, когда они покидают сервис.

**Динамическое создание объектов** — это выделение памяти под объект из специальной области памяти, называемой "куча" (`heap`), во время выполнения программы. Программист сам решает, когда создать объект и когда его уничтожить. В C++ для этого используются операторы `new` и `delete`.

Понимание динамической памяти и жизненного цикла объектов критически важно для написания эффективных и надежных программ. Неправильное управление памятью приводит к **утечкам памяти** (память не освобождается) и **висячим указателям** (указатели ссылаются на уже удаленную память), что является источником трудноуловимых ошибок.

На этом занятии мы подробно изучим, как создавать и удалять объекты в динамической памяти, как отслеживать их жизненный цикл и как избежать типичных ошибок.

## Статическое и динамическое создание объектов

В C++ существует два основных способа создания объектов, которые принципиально различаются по месту выделения памяти и времени жизни.

### Статическое создание (на стеке):
- Память выделяется в области стека
- Время жизни объекта определяется областью видимости
- Объект автоматически уничтожается при выходе из области видимости
- Используется синтаксис: `ClassName objectName`; или `ClassName objectName(args)`;

### Динамическое создание (в куче):
- Память выделяется в области кучи (`heap`)
- Время жизни объекта управляется вручную программистом
- Объект существует до тех пор, пока не будет явно удален оператором `delete`
- Используется синтаксис: `ClassName* ptr = new ClassName`; или `ClassName* ptr = new ClassName(args)`;

### ПРИМЕР 1 (Статическое создание)

```cpp
#include <iostream>

class Point {
private:
    int x, y;
public:
    Point(int xVal = 0, int yVal = 0) : x(xVal), y(yVal) {
        std::cout << "Конструктор Point(" << x << ", " << y << ") вызван" << std::endl;
    }
    
    ~Point() {
        std::cout << "Деструктор Point(" << x << ", " << y << ") вызван" << std::endl;
    }
    
    void display() const {
        std::cout << "Точка: (" << x << ", " << y << ")" << std::endl;
    }
};

int main() {
    std::cout << "Начало main()" << std::endl;
    
    { // Начало области видимости
        Point p1(10, 20); // Статическое создание
        p1.display();
    } // Конец области видимости - p1 автоматически уничтожается
    
    std::cout << "Между блоками" << std::endl;
    
    { // Другой блок
        Point p2(30, 40); // Еще один статический объект
        p2.display();
    } // p2 уничтожается
    
    std::cout << "Конец main()" << std::endl;
    return 0;
}
```

### ПРИМЕР 2 (Динамическое создание)

```cpp
#include <iostream>

class Point {
    // Тело класса полностью совпадает с примером 1
private:
    int x, y;
public:
    Point(int xVal = 0, int yVal = 0) : x(xVal), y(yVal) {
        std::cout << "Конструктор Point(" << x << ", " << y << ") вызван" << std::endl;
    }
    
    ~Point() {
        std::cout << "Деструктор Point(" << x << ", " << y << ") вызван" << std::endl;
    }
    
    void display() const {
        std::cout << "Точка: (" << x << ", " << y << ")" << std::endl;
    }
};

int main() {
    std::cout << "Начало main()" << std::endl;
    
    // Динамическое создание объекта
    Point* pPtr = new Point(100, 200);
    pPtr->display();
    
    // Объект продолжает существовать, даже если мы выходим из области видимости указателя
    {
        Point* innerPtr = new Point(300, 400);
        innerPtr->display();
        // Здесь НЕ вызывается delete для innerPtr - утечка памяти!
    } // Указатель innerPtr уничтожается, но объект в куче продолжает существовать
    
    std::cout << "До удаления pPtr" << std::endl;
    delete pPtr; // Только теперь объект (100, 200) уничтожается
    
    std::cout << "Конец main()" << std::endl;
    // Объект (300, 400) так и не был удален - утечка памяти
    return 0;
}
```

## Операторы `new` и `delete`
Для работы с динамической памятью в C++ используются операторы `new` и `delete`.

**Оператор `new`** выполняет две задачи:
- Выделяет достаточный объем памяти в куче для размещения объекта
- Вызывает конструктор объекта для инициализации этой памяти
**Оператор `delete`** также выполняет две задачи:
- Вызывает деструктор объекта для освобождения занятых ресурсов
- Освобождает память, занятую объектом в куче

Для массивов объектов используются операторы `new[]` и `delete[]`.

### ПРИМЕР 3 (new и delete для одиночного объекта)

```cpp
#include <iostream>
#include <string>

class User {
private:
    std::string name;
    int age;
public:
    User() : name("Аноним"), age(0) {
        std::cout << "Конструктор User() для " << name << std::endl;
    }
    
    User(std::string userName, int userAge) : name(userName), age(userAge) {
        std::cout << "Конструктор User(параметры) для " << name << std::endl;
    }
    
    ~User() {
        std::cout << "Деструктор User для " << name << std::endl;
    }
    
    void introduce() const {
        std::cout << "Привет! Я " << name << ", мне " << age << " лет." << std::endl;
    }
};

int main() {
    // Выделение памяти и вызов конструктора по умолчанию
    User* user1 = new User;
    user1->introduce();
    
    // Выделение памяти и вызов конструктора с параметрами
    User* user2 = new User("Алиса", 25);
    user2->introduce();
    
    // Обязательно освобождаем память!
    delete user1;
    delete user2;
    
    // После удаления указатели становятся висячими, их лучше обнулить
    user1 = nullptr;
    user2 = nullptr;
    
    return 0;
}
```

### ПРИМЕР 4 (`new[]` и `delete[]` для массивов объектов)

```cpp
#include <iostream>

class Item {
private:
    int id;
public:
    Item() {
        id = 0;
        std::cout << "Конструктор Item() по умолчанию" << std::endl;
    }
    
    Item(int itemId) : id(itemId) {
        std::cout << "Конструктор Item(" << id << ")" << std::endl;
    }
    
    ~Item() {
        std::cout << "Деструктор Item для id=" << id << std::endl;
    }
    
    void setId(int itemId) { id = itemId; }
    int getId() const { return id; }
};

int main() {
    // Создание массива объектов в динамической памяти
    // Для каждого элемента будет вызван конструктор по умолчанию
    std::cout << "Создание массива из 3 объектов Item:" << std::endl;
    Item* items = new Item[3];
    
    // Инициализация элементов массива
    for (int i = 0; i < 3; ++i) {
        items[i].setId(i + 1);
    }
    
    std::cout << "Содержимое массива:" << std::endl;
    for (int i = 0; i < 3; ++i) {
        std::cout << "items[" << i << "].id = " << items[i].getId() << std::endl;
    }
    
    // ВАЖНО! Для освобождения массива используется delete[]
    std::cout << "Удаление массива:" << std::endl;
    delete[] items; // Вызовет деструктор для каждого элемента
    
    return 0;
}
```

## Жизненный цикл объекта
Жизненный цикл объекта в C++ состоит из нескольких четко определенных этапов:
- **Выделение памяти** — под объект резервируется место в памяти (стек или куча)
- **Инициализация** — вызывается конструктор, создается целостное состояние объекта
- **Использование** — объект выполняет свои функции, с ним взаимодействуют другие части программы
- **Уничтожение** — вызывается деструктор для освобождения захваченных ресурсов
- **Освобождение памяти** — память, занятая объектом, помечается как свободная

Для объектов в стеке этапы 1 и 5 управляются автоматически. Для объектов в куче этапы 1 и 5 выполняются операторами `new` и `delete`.

### ПРИМЕР 5 (Отслеживание жизненного цикла)

```cpp
#include <iostream>
#include <string>

class LifecycleDemo {
private:
    std::string name;
    static int objectCount; // Статическое поле для подсчета объектов
public:
    // Конструктор
    LifecycleDemo(std::string objName) : name(objName) {
        objectCount++;
        std::cout << "+++ Конструктор: объект \"" << name << "\" создан. "
                  << "Всего объектов: " << objectCount << std::endl;
    }
    
    // Деструктор
    ~LifecycleDemo() {
        objectCount--;
        std::cout << "--- Деструктор: объект \"" << name << "\" уничтожен. "
                  << "Осталось объектов: " << objectCount << std::endl;
    }
    
    // Конструктор копирования
    LifecycleDemo(const LifecycleDemo& other) : name(other.name + "_copy") {
        objectCount++;
        std::cout << "### Конструктор копирования: создана копия \"" << name 
                  << "\". Всего объектов: " << objectCount << std::endl;
    }
    
    void doSomething() const {
        std::cout << "Объект \"" << name << "\" выполняет действие." << std::endl;
    }
};

// Инициализация статического поля
int LifecycleDemo::objectCount = 0;

// Функция, принимающая объект по значению (создается копия)
void processObject(LifecycleDemo obj) {
    std::cout << "Функция processObject() начала работу." << std::endl;
    obj.doSomething();
    std::cout << "Функция processObject() завершает работу." << std::endl;
} // Здесь деструктор уничтожит локальную копию obj

// Функция, возвращающая объект
LifecycleDemo createObject() {
    std::cout << "Функция createObject() начала работу." << std::endl;
    LifecycleDemo temp("Временный");
    std::cout << "Функция createObject() завершает работу." << std::endl;
    return temp; // Возврат по значению (может быть оптимизирован компилятором)
}

int main() {
    std::cout << "Начало программы." << std::endl;
    
    // Статическое создание
    LifecycleDemo static_obj("Статический");
    
    // Динамическое создание
    LifecycleDemo* dynamic_obj = new LifecycleDemo("Динамический");
    
    // Вызов функции с передачей по значению
    std::cout << "\nВызов processObject():" << std::endl;
    processObject(static_obj);
    
    // Вызов функции, возвращающей объект
    std::cout << "\nВызов createObject():" << std::endl;
    LifecycleDemo returned_obj = createObject();
    
    // Использование объектов
    std::cout << "\nИспользование объектов:" << std::endl;
    static_obj.doSomething();
    dynamic_obj->doSomething();
    returned_obj.doSomething();
    
    // Обязательно удаляем динамический объект
    std::cout << "\nУдаление динамического объекта:" << std::endl;
    delete dynamic_obj;
    
    std::cout << "\nКонец программы." << std::endl;
    // Здесь автоматически уничтожатся static_obj и returned_obj
    return 0;
}
```

## Утечки памяти и правила работы с динамической памятью
**Утечка памяти** происходит, когда программа теряет возможность освободить ранее выделенную память. Это случается, если:
- Указатель на выделенную память перезаписывается до вызова `delete`
- Указатель выходит из области видимости, а `delete` не был вызван
- В программе возникает исключение до вызова `delete`
- Утечки памяти приводят к тому, что программа потребляет все больше памяти и в конце концов может исчерпать доступные ресурсы системы.

### ПРИМЕР 6 (Утечка памяти)

```cpp
#include <iostream>

void leakExample() {
    int* ptr = new int(42); // Выделяем память
    std::cout << "Значение: " << *ptr << std::endl;
    // Забываем вызвать delete!
    // Память утекла
} // ptr выходит из области видимости, адрес выделенной памяти потерян навсегда

void overwriteLeak() {
    int* ptr = new int(100);
    ptr = new int(200); // Перезаписываем указатель, не удалив старую память
    // Первый объект (100) больше недоступен - утечка!
    delete ptr; // Удаляется только второй объект
}

int main() {
    std::cout << "Демонстрация утечек памяти:" << std::endl;
    
    for (int i = 0; i < 5; ++i) {
        leakExample();
    }
    
    overwriteLeak();
    
    std::cout << "Программа завершена. Память не освобождена полностью." << std::endl;
    return 0;
}
```

### 🌟 Задача 8. Пул объектов (Object Pool)

Создайте класс `ConnectionPool`, который реализует паттерн "Пул объектов". Это класс, который заранее создает фиксированное количество объектов и управляет их переиспользованием, чтобы избежать затрат на частое создание и удаление.

**Требования к классу `Connection` (внутренний класс или отдельный класс):**

1.	Поля:
-	`int id` — уникальный идентификатор соединения
-	`bool inUse` — флаг, используется ли соединение в данный момент
2.	Конструктор присваивает `id` и устанавливает `inUse = false`
3.	Метод `connect()` — устанавливает `inUse = true`, выводит: `"Соединение #[id] активировано"`
4.	Метод `disconnect()` — устанавливает `inUse = false`, выводит: `"Соединение #[id] деактивировано"`
5.	Метод `isAvailable()` — возвращает !inUse
Требования к классу `ConnectionPool`:
1.	Поля:
-	`Connection** pool` — динамический массив указателей на соединения
-	`int poolSize` — размер пула
2.	Конструктор с параметром (размер пула):
-	Выделяет память под массив указателей
-	Создает указанное количество объектов `Connection` в динамической памяти
-	Сохраняет указатели на них в массив
-	Выводит: `"Создан пул из X соединений"`
3.	Деструктор:
-	Проходит по всем элементам пула и удаляет объекты `Connection`
-	Освобождает память массива указателей
-	Выводит: `"Пул соединений уничтожен"`
4.	Метод `acquireConnection()`:
-	Ищет первое доступное соединение (`с inUse == false`)
-	Если находит, вызывает его `connect()` и возвращает указатель на него
-	Если не находит, выводит: `"Нет доступных соединений!"` и возвращает `nullptr`
5.	Метод `releaseConnection(Connection* conn)`:
-	Вызывает `disconnect()` для указанного соединения
6.	Метод `printStatus()`:
-	Выводит статус всех соединений (`ID` и флаг использования)
7.	В функции `main()`:
-	Создайте пул из `3` соединений
-	Запросите `2` соединения, используйте их, затем верните
-	Запросите `3` соединения (все доступные)
-	Попробуйте запросить четвертое — должно вернуть `nullptr`
-	Верните одно соединение и снова запросите — должно получиться
-	В конце программы убедитесь, что деструктор корректно удалил все объекты
