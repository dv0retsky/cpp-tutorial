### Что такое инкапсуляция в ООП? 🤔

**Инкапсуляция** — это принцип объектно-ориентированного программирования (ООП), который предполагает объединение данных с методами, которые с ними работают, в единое целое. Еще иногда говорят, что данные помещаются в «оболочку», так как инкапсуляция — это также способ скрыть внутреннюю реализацию и ограничить доступ к составляющим объекта так, чтобы избежать их нежелательного чтения, изменения и удаления.


В **C++** инкапсуляция реализуется также с помощью модификаторов доступа и методов для работы со свойствами.

Рассмотрим пример:

```cpp
#include <iostream> 
#include <string>   
using namespace std;

class Book
{
  private:
    // Инкапсуляция свойств названия и автора книги
    string title;
    string author;

  public:
    // Конструктор
    Book(string title, string author)
    {
        this->title = title;
        this->author = author;
    }

    // Сеттер для установки названия книги
    void setTitle(string title)
    {
        this->title = title;
    }

    // Геттер для получения названия книги
    string getTitle()
    {
        return title;
    }

    // Сеттер для установки автора книги
    void setAuthor(string author)
    {
        this->author = author;
    }

    // Геттер для получения автора книги
    string getAuthor()
    {
        return author;
    }
};

int main()
{
    // Создание объекта класса Книга
    Book book("Мастер и Маргарита", "Михаил Булгаков");

    // Использование геттеров
    cout << "Название: " << book.getTitle() << endl;
    cout << "Автор: " << book.getAuthor() << endl;

    // Установка значений свойств название и автор
    book.setTitle("Руслан и Людмила");
    book.setAuthor("Александр Пушкин");

    // Вывод новых значений на экран с использованием геттеров
    cout << "Название: " << book.getTitle() << endl;
    cout << "Автор: " << book.getAuthor() << endl;

    return 0;
}

// Вывод: Название: Мастер и Маргарита
    // Автор: Михаил Булгаков
    // Название: Руслан и Людмила
    // Автор: Александр Пушкин
```

## 🐙 Отличие инкапсуляции от сокрытия

Иногда инкапсуляцию и сокрытие считают синонимами и объединяют в один принцип ООП, но на самом деле эти понятия имеют существенные отличия. Сокрытие данных предполагает обеспечение их безопасности путем частичного или полного ограничения доступа к составляющим объекта. Инкапсуляция же предполагает объединение данных и их методов так, чтобы они образовывали единое целое, что позволяет взаимодействовать с объектом, не вмешиваясь в его внутреннее устройство.

При этом инкапсуляция и сокрытие связаны между собой. Инкапсуляция предусматривает создание интерфейсов для обеспечения контроля доступа к данным, в то время как сокрытие обеспечивает их безопасность. Можно сказать, что инкапсуляция включает в себя сокрытие, при этом само сокрытие не является инкапсуляцией.

Реализация этих принципов отличается от языка к языку. Где-то эти принципы четко разграничены, где-то нет смысла реализовывать инкапсуляцию без сокрытия, а, например, в Python этот принцип реализуется без сокрытия, полагаясь на соглашение между разработчиками.

Рассмотрим преимущества инкапсуляции:

- этот принцип позволяет управлять доступом к данным и скрывать детали реализации классов;
- повышение гибкости кода: в код легко вносить изменения, которые не повлияют на работу оставшейся части программы, также код легче переиспользовать;
- повышение читаемости кода, упрощение его поддержки, тестирования и отладки.

## 🎀 Как выглядит реализация инкапсуляции

Конкретный вид реализации этого принципа может отличаться в разных языках, но суть остается прежней: этот принцип в ООП основан на создании классов и объектов, которые и представляют из себя «оболочку» для данных (свойств) и функций (методов).

Класс выступает как абстрактное описание, которое определяет общую структуру и поведение будущих объектов этого класса. Объекты же, в свою очередь, объединяют свойства и методы в целостную сущность, тем самым защищая свое содержимое от внешних вмешательств.

Таким образом, реализация инкапсуляции сводится к созданию класса и ограничению уровня доступа к его свойствам, а также определению методов, которые позволят контролировать взаимодействие со свойствами извне.

## 🌸 Как разработчики инкапсулируют данные

Конкретная реализация этого принципа зависит не только от языка программирования, но и от разработчика. Тем не менее есть основные методы и техники, позволяющие инкапсулировать данные:

- Использование модификаторов доступа. Это один из наиболее часто используемых методов инкапсуляции данных, но доступный не в каждом языке программирования. Его суть состоит в ограничении доступа к данным через использование модификаторов доступа: `public`, `private` и `protected`. `Public` подразумевает, что объект или метод доступен из любого места программы — это удобно, но может нарушать инкапсуляцию и безопасность данных. `Private` делает содержимое объекта доступным только внутри этого объекта — например, методы не могут вызываться каким-то кодом извне. `Protected` же подразумевает, что доступ к содержимому объекта разрешен самому объекту и его потомкам. Существуют и другие модификаторы доступа, а также ключевые слова, например, `sealed` и `abstract`.

- Реализация геттеров и сеттеров. Это специальные методы, название которых обычно начинается со слов `get` и `set`. Эти методы обеспечивают контролируемый доступ к свойствам объекта без вмешательства в его внутреннее устройство — геттеры возвращают текущее значение свойства объекта, а сеттеры изменяют значения свойств.

- Реализация свойств и методов. Свойства (переменные, определенные внутри класса, которые хранят данные) и методы (функции, определенные внутри класса, которые выполняют действия) — это ключевые компоненты классов в ООП, и они тоже играют немаловажную роль при реализации инкапсуляции. Доступ к содержимому свойств может быть ограничен, например, с помощью модификаторов доступа, а методы могут вызываться только для объектов, в которых они определены.

Также нельзя забывать о том, что для грамотной реализации этого принципа специалист должен обладать знаниями и навыками проектирования классов.

## 🎟️ Закрепим информацию! 

**Инкапсуляция** — это одна из ключевых концепций объектно-ориентированного программирования (ООП), которая позволяет объединять данные и методы, работающие с этими данными, в одном классе, а также ограничивать доступ к внутренним данным извне. Основная цель инкапсуляции — сокрытие внутренней реализации объекта и предоставление контролируемого интерфейса для взаимодействия с ним. Это делает код более безопасным, удобным для использования и поддерживаемым.

Чтобы лучше понять инкапсуляцию, рассмотрим пример из жизни. Представьте, что у вас есть автомобиль. Водитель не знает, как устроен двигатель, как работает коробка передач или топливная система. Однако он может управлять автомобилем с помощью руля, педалей и рычагов. В этом случае автомобиль — это объект, а его внутренние механизмы — это скрытые данные, доступ к которым ограничен. Руль, педали и рычаги — это интерфейс, который позволяет водителю взаимодействовать с автомобилем. Именно так работает инкапсуляция в программировании: данные скрыты, а взаимодействие с объектом происходит через публичные методы.

На занятии мы изучим, как реализовать инкапсуляцию в C++ с помощью классов, модификаторов доступа, конструкторов и деструкторов. Эти знания помогут вам создавать безопасные и удобные в использовании объекты.

## 📕 Классы в C++

**Класс** — это пользовательский тип данных, который объединяет данные (поля) и методы (функции), работающие с этими данными. Класс можно рассматривать как чертеж, на основе которого создаются объекты. Например, если класс — это чертеж автомобиля, то объект — это конкретный автомобиль, созданный по этому чертежу.

В C++ класс объявляется с помощью ключевого слова `class`. Внутри класса определяются поля (данные) и методы (функции). Например, рассмотрим класс `Car`, который представляет автомобиль:

```cpp
#include <iostream>
#include <string>

// Объявление класса (чертеж автомобиля)
class Car {
private:
    // Приватные поля (скрытые данные)
    int speed;
    std::string model;

public:
    // Публичные методы (интерфейс для управления)

    // Конструктор для инициализации
    Car(std::string carModel) {
        model = carModel;
        speed = 0;
        std::cout << "Создан автомобиль " << model << "! Начальная скорость: " << speed << std::endl;
    }

    // Увеличить скорость
    void accelerate(int value) {
        if (value > 0) {
            speed += value;
            std::cout << "Автомобиль " << model << " ускорился. Текущая скорость: " << speed << std::endl;
        }
    }

    // Уменьшить скорость
    void brake(int value) {
        if (value > 0 && speed - value >= 0) {
            speed -= value;
            std::cout << "Автомобиль " << model << " замедлился. Текущая скорость: " << speed << std::endl;
        } else {
            std::cout << "Нельзя уменьшить скорость ниже 0!" << std::endl;
        }
    }

    // Получить текущую скорость (метод-геттер)
    int getSpeed() const {
        return speed;
    }

    // Получить модель (метод-геттер)
    std::string getModel() const {
        return model;
    }
};

// Точка входа в программу (функция main будет добавлена в следующем примере)
// int main() {
//     return 0;
// }
```

В этом примере:
- Поле `speed` хранит текущую скорость автомобиля.
- Метод `accelerate` увеличивает скорость на указанное значение.
- Метод `brake` уменьшает скорость на указанное значение.
- Метод `getSpeed` возвращает текущую скорость.

После объявления класса можно создавать объекты этого класса. Например:

```cpp
#include <iostream>
#include <string>

class Car {
    // ... (Здесь полностью повторяется тело класса из Примера 1) ...
private:
    int speed;
    std::string model;

public:
    Car(std::string carModel) {
        model = carModel;
        speed = 0;
        std::cout << "Создан автомобиль " << model << "! Начальная скорость: " << speed << std::endl;
    }

    void accelerate(int value) {
        if (value > 0) {
            speed += value;
            std::cout << "Автомобиль " << model << " ускорился. Текущая скорость: " << speed << std::endl;
        }
    }

    void brake(int value) {
        if (value > 0 && speed - value >= 0) {
            speed -= value;
            std::cout << "Автомобиль " << model << " замедлился. Текущая скорость: " << speed << std::endl;
        } else {
            std::cout << "Нельзя уменьшить скорость ниже 0!" << std::endl;
        }
    }

    int getSpeed() const {
        return speed;
    }
};

int main() {
    // Создание объекта (конкретного автомобиля)
    Car myCar("Tesla Model 3");

    // Управление автомобилем через публичные методы
    myCar.accelerate(50);
    myCar.accelerate(30);
    myCar.brake(20);

    // Получение данных через метод (геттер)
    std::cout << "Финальная скорость автомобиля " << myCar.getModel() << ": " << myCar.getSpeed() << std::endl;

    // Следующая строка вызвала бы ошибку компиляции, так как speed - private
    // myCar.speed = 100; // ОШИБКА! Нельзя обратиться напрямую.

    return 0;
}
```

Здесь мы создаем объект `myCar` класса `Car` и управляем его скоростью с помощью методов `accelerate` и `brake`. Обратите внимание, что поле `speed` недоступно напрямую извне класса, так как оно объявлено как `private`. Это и есть пример инкапсуляции: данные скрыты, а взаимодействие с объектом происходит через публичные методы.

## 🍒 Модификаторы доступа

**Модификаторы доступа** — это ключевые слова, которые определяют, кто может получить доступ к полям и методам класса. В **C++** существует три модификатора доступа:

- `private`: Поля и методы доступны только внутри класса.
- `protected`: Поля и методы доступны внутри класса и в производных классах.
- `public`: Поля и методы доступны из любого места программы.

Рассмотрим пример класса `BankAccount`, который демонстрирует использование модификаторов доступа:

```cpp
#include <iostream>
#include <string>

class BankAccount {
private:
    // Приватное поле: доступно только внутри класса
    double balance;
    std::string accountNumber;

    // Приватный метод для внутренней проверки (скрыт от пользователя)
    bool isValidAmount(double amount) const {
        return amount > 0;
    }

public:
    // Публичный конструктор
    BankAccount(std::string accNum, double initialBalance) {
        accountNumber = accNum;
        if (initialBalance >= 0) {
            balance = initialBalance;
        } else {
            balance = 0;
            std::cout << "Начальный баланс не может быть отрицательным. Установлен 0." << std::endl;
        }
        std::cout << "Счет " << accountNumber << " создан. Баланс: " << balance << " руб." << std::endl;
    }

    // Публичный метод для внесения денег
    void deposit(double amount) {
        if (isValidAmount(amount)) {
            balance += amount;
            std::cout << "Внесено " << amount << " руб. Текущий баланс: " << balance << " руб." << std::endl;
        } else {
            std::cout << "Сумма для внесения должна быть положительной!" << std::endl;
        }
    }

    // Публичный метод для снятия денег (с проверкой)
    void withdraw(double amount) {
        if (isValidAmount(amount) && amount <= balance) {
            balance -= amount;
            std::cout << "Снято " << amount << " руб. Текущий баланс: " << balance << " руб." << std::endl;
        } else {
            std::cout << "Недостаточно средств или неверная сумма!" << std::endl;
        }
    }

    // Публичный метод для проверки баланса (геттер)
    double getBalance() const {
        return balance;
    }

    // Публичный метод для получения номера счета (геттер)
    std::string getAccountNumber() const {
        return accountNumber;
    }
};

int main() {
    BankAccount myAccount("40817810099910000123", 5000);

    myAccount.deposit(1500);
    myAccount.withdraw(2000);
    myAccount.withdraw(10000); // Попытка снять больше, чем есть

    std::cout << "Итоговый баланс счета " << myAccount.getAccountNumber() << ": " << myAccount.getBalance() << " руб." << std::endl;

    // Следующая строка вызовет ошибку компиляции
    // myAccount.balance = 1000000; // ОШИБКА! balance - private

    return 0;
}
```

В этом примере:

- Поле `balance` скрыто от внешнего доступа с помощью модификатора `private`.
- Методы `deposit`, `withdraw` и `getBalance` объявлены как `public`, что позволяет взаимодействовать с объектом извне.
- Использование модификаторов доступа позволяет контролировать, какие данные и методы доступны для внешнего использования, а какие — нет. 

Это делает код более безопасным и защищенным от ошибок.

## 🍇 Конструкторы и деструкторы

**Конструктор** — это специальный метод, который вызывается при создании объекта. Он используется для инициализации полей объекта. В **C++** конструктор имеет то же имя, что и класс, и не имеет возвращаемого типа. Например:

```cpp
#include <iostream>
#include <string>

class Person {
private:
    std::string name;
    int age;

public:
    // Конструктор (вызывается автоматически при создании объекта)
    Person(std::string personName, int personAge) {
        name = personName;
        if (personAge > 0 && personAge < 150) {
            age = personAge;
        } else {
            age = 0;
            std::cout << "Некорректный возраст! Установлен 0." << std::endl;
        }
        std::cout << "Конструктор вызван. Создан объект Person для " << name << std::endl;
    }

    // Метод для вывода информации
    void introduce() const {
        std::cout << "Привет! Меня зовут " << name << ", мне " << age << " лет." << std::endl;
    }

    // Геттер для имени
    std::string getName() const {
        return name;
    }
};

int main() {
    // Конструктор вызывается в момент создания объекта
    Person person1("Alice", 30);
    Person person2("Bob", 25);

    person1.introduce();
    person2.introduce();

    return 0;
}
```

В этом примере конструктор инициализирует поля `name` и `age` при создании объекта `person`.

**Деструктор** — это специальный метод, который вызывается при уничтожении объекта. Он используется для освобождения ресурсов, которые использовались объектом. 

**Деструктор** имеет то же имя, что и класс, но начинается с символа `~`. Например:

```cpp
#include <iostream>

class SimpleArray {
private:
    int* data; // Указатель на массив в куче (динамическая память)
    int size;

public:
    // Конструктор: выделяем память
    SimpleArray(int arrSize) {
        size = arrSize;
        data = new int[size]; // Запрашиваем память у системы
        std::cout << "Конструктор: выделена память для " << size << " элементов." << std::endl;

        // Для примера заполняем массив числами
        for (int i = 0; i < size; ++i) {
            data[i] = i * 10;
        }
    }

    // Метод для вывода элементов
    void print() const {
        for (int i = 0; i < size; ++i) {
            std::cout << data[i] << " ";
        }
        std::cout << std::endl;
    }

    // Деструктор: освобождаем память (вызывается автоматически)
    ~SimpleArray() {
        delete[] data; // Возвращаем память системе
        std::cout << "Деструктор: память освобождена." << std::endl;
    }
};

// Функция для демонстрации времени жизни объекта
void demoFunction() {
    std::cout << "--- Вход в функцию demoFunction ---" << std::endl;
    SimpleArray localArray(5); // Объект создается здесь
    localArray.print();
    std::cout << "--- Выход из функции demoFunction (объект localArray будет уничтожен) ---" << std::endl;
} // Здесь объект localArray выходит из области видимости, и деструктор вызывается автоматически

int main() {
    std::cout << "Начало работы main()" << std::endl;

    demoFunction(); // Вызов функции

    // Создаем объект в main
    SimpleArray mainArray(3);
    mainArray.print();

    std::cout << "Конец работы main() (объект mainArray будет уничтожен)" << std::endl;
    return 0;
} // Здесь вызывается деструктор для mainArray
```

В этом примере деструктор выводит сообщение при уничтожении объекта.
